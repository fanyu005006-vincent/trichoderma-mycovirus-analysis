# Potential host network analysis

This clean package contains the two core analyses used to support the
manuscript's potential host network model:

1. A saved BLASTp and NCBI `source/host` workflow that ranks host association
   hypotheses for each observed viral query profile.
2. A secondary AlphaFold 3 and TM-align comparison that quantifies structural
   concordance between selected RdRp region models.

The package is intentionally limited to these reviewer-facing analyses. The
older raw read workflow, early host filter, figure redraw scripts and local
generated results are not included.

## Inputs

`data/host_prediction_inputs/` contains the saved BLASTp text files.
`data/reference_host_metadata.csv` supplies the NCBI GenBank `source/host`
qualifier for each matched accession. Host identity is never inferred from a
virus name. Tick derived environmental records and unresolved labels are
excluded. Two explicit literature reported hosts use the curated
`reported_host_override` column: grapevine records are represented as
`Vitis vinifera`, and the Yellow silver pine record is retained as `Yellow
silver pine`. These overrides preserve host information reported in the virus
literature even when the NCBI `source/host` field is generic.

`data/af3_structure_manifest.csv` identifies the supplied AlphaFold 3 models;
the corresponding mmCIF files are in `data/af3_structures/`. These are RdRp
region models and are used only as secondary structural support. The CV2
manifest entry reuses the CV1 model because the two RdRp sequences are
identical, and is explicitly labelled as copied evidence.

Each observed query is an independent analysis profile. TsCV2 is explicitly
derived from the TsCV1 input because their RdRp sequences are identical in
the manuscript; the copied records are marked
`copied_from_CV1_identical_RdRp`. This is a transparent data derivation, not
an automatic merge or a sequence similarity inference. Other queries are not
copied or merged.

## Model and statistics

For each query, the workflow applies E-value `<= 1e-5`, identity `>= 25%` and
query span coverage `>= 40%`. It retains the strongest accession hit, then the
strongest representative for each host genus, and ranks up to ten distinct
host genera. The similarity weighted evidence score is:

```text
bitscore × identity × query coverage
```

Ten thousand weighted bootstrap resamples report empirical support for the
highest supported host category within each query. Five thousand permutations
shuffle host category labels within viral family. Empirical P values use the
plus one correction and are adjusted across query by category tests with the
Benjamini Hochberg procedure.

These outputs rank host association hypotheses. They are not infection
probabilities, complete natural host ranges, or evidence of autonomous viral
replication.

## Run

From the package root:

```bash
python -m pip install -r requirements.txt
python run_analysis.py
```

For the standard library host model only:

```bash
python run_analysis.py --skip-structure
```

The supplied AF3 files are not regenerated by the script. The structural step
compares the supplied models with TM-align and leaves network edges without a
matching model explicitly marked as unavailable.

## Outputs

`results/host_network/` contains:

- `raw_blast_host_evidence.csv`: every parsed BLASTp hit, including the
  explicit source host, threshold status, exclusion reason and whether the
  record entered the quantitative network.
- `potential_host_network_edges.csv`: up to ten ranked host genera per query.
- `accepted_accession_evidence.csv`: accession level evidence after threshold
  and host metadata filtering.
- `host_category_support.csv`: weighted mass and 10,000 replicate support.
- `host_category_enrichment.csv`: family stratified permutation P values and
  Benjamini Hochberg adjusted values.
- `host_taxon_recurrence.csv`: recurrence of host genera across profiles.
- `candidate_manifest.csv`: observed query profiles and inclusion status.
- `analysis_summary.json`: thresholds, replicate counts and interpretation
  boundaries.

`results/structural_support/` contains:

- `af3_structure_pair_scores.csv`: pairwise TM-align metrics and the fixed
  structural support score.
- `potential_host_network_edges_with_af3.csv`: network edges annotated with
  structure support when an exact accession mapping exists.
- `structural_support_summary.json`: model counts and score definition.

## Reproducibility boundaries

The source BLASTp text and metadata are included, so the host network can be
recomputed without a live database query. The AF3 models are supplied inputs;
the package does not claim to reproduce the original AF3 prediction job. The
structural score is descriptive support for the potential host network, not a
host prediction engine by itself.
